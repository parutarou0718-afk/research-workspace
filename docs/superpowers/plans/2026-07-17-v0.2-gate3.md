# v0.2 Gate 3 Protected CRUD, Relations, Audit, Undo, and Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver Gate 3 only: protected Paper/Idea/Submission editing, formal PaperVersion membership and DAG decisions, immutable command/audit history, field-aware compensating undo, verified pre-command recovery, and the approved interactive UI.

**Architecture:** Gate 3 is additive over the frozen Gate 2 facts and interfaces. A raw user request is authorized and canonicalized into one immutable `ApplicationCommand`; a verified two-slot SQLite recovery point is created before the short domain transaction; the coordinator then atomically commits domain rows, per-entity `AuditChange` rows, and minimal DomainEvent 2.0 rows. PaperVersion membership and formal relations are explicit audited facts, while Gate 2 candidate identity, evidence, rules, scheduling, and monitoring remain unchanged.

**Tech Stack:** Python 3.12, PySide6, SQLAlchemy 2.0, Alembic, SQLite Online Backup API, RFC 8785 canonical JSON, SHA-256, jsonschema Draft 2020-12, pytest, pytest-qt, and pytest-socket.

**Normative source:** `docs/superpowers/specs/2026-07-16-v0.2-deterministic-design.md` at commit `f55fb98ed6b8723507fc4564ba614dca0aa249a8`.

**Required baseline:** Frozen Gate 2 checkpoint `2a5f033aa721e4ad8ef75b0ecc3eac127cf3d477`, Alembic revision `0003_gate2_monitoring`, with `866 passed`.

## Global Constraints

- Gate 2 is frozen. Its contracts, migration, candidate identity, detector revision, R1-R5 rules, comparison scheduler, monitoring semantics, and existing facts may change only for a separately approved critical bug, security defect, or data-corruption defect.
- Implement by forward addition. Gate 3 must not reinterpret, rebuild, overwrite, or silently “upgrade” a Gate 2 candidate, evidence snapshot, monitoring event, observation, or reconciliation fact.
- Implement Gate 3 only. Do not add Gate 4 backup/restore/export, OCR, AI, Agent, TaskExecutor, arbitrary Task execution, permanent deletion, force undo, redo, undo-of-undo, SubmissionRound, or CRUD for Note/Conference/Grant.
- Every user-visible domain write is one `ApplicationCommand`, one authorization decision, one verified pre-command recovery point, and one domain transaction.
- Domain facts, per-entity `AuditChange` rows, and business DomainEvent 2.0 rows commit in the same SQLite transaction. Failed commands commit none of them.
- Recovery points protect against write-time disaster; compensating commands implement ordinary undo. A recovery image is never used as the normal undo mechanism.
- Updates use atomic compare-and-update with expected `row_version`. Zero affected rows returns `CONCURRENT_MODIFICATION`.
- Same idempotency key plus same RFC 8785 request fingerprint returns the prior result; the same key with different content returns `COMMAND_IDEMPOTENCY_CONFLICT`.
- Soft deletion is conservative and non-cascading. Permanent deletion and silent relation detachment are absent.
- Formal PaperVersion succession always uses `source=later PaperVersion`, `target=earlier PaperVersion`; active edges form a per-Paper DAG.
- Gate 2 candidates remain immutable evidence. Gate 3 changes only approved lifecycle/decision fields and creates separate formal facts.
- Workers own no SQLAlchemy Session, repository, QWidget, Qt model, formal DomainEvent publication, arbitrary path, permission token, or network capability.
- `PermissionContext` remains an audit snapshot for `user|system`, never a reusable credential. `agent` and `task_executor` are rejected before context construction.
- All built-in workflows keep `network_allowed=false`.
- Every long-lived layout and dialog is owned by a `.ui` file. Python controllers remain thin and never construct layouts imperatively.
- Tests first fail for a genuine unmet requirement. Collection errors count as RED only when caused by an approved missing module; test-harness errors are corrected and rerun before recording RED.
- Do not manufacture RED, weaken a test, add a skip/xfail, or alter supplied files to obtain GREEN.
- After every GREEN Task, run its focused command and `uv run pytest -q`.
- Every Task produces one independently revertible commit. A Checkpoint may contain multiple Task commits, but commits are never squashed for review.
- `REL-GATE-001` remains `BLOCKED_BY_ENVIRONMENT` and blocks only public packaged release.

## Fixed Interface Ledger

Implementations may use canonical JSON bytes or deeply immutable value objects, but must preserve these names and meanings:

```python
@dataclass(frozen=True)
class RawCommandEnvelope:
    command_id: UUID
    command_type: str
    contract_version: Literal["1.0"]
    idempotency_key: str
    actor_type: str
    actor_id: str | None
    workspace_id: UUID
    requested_at: datetime
    request_payload: bytes  # RFC 8785 canonical JSON

@dataclass(frozen=True)
class CommandPlan:
    command_id: UUID
    command_type: str
    idempotency_key: str
    request_fingerprint: str
    permission_context: bytes
    entity_scopes: tuple[tuple[str, UUID], ...]
    expected_versions: tuple[tuple[str, UUID, int], ...]
    canonical_request: bytes
    protected: bool

@dataclass(frozen=True)
class RecoveryPlan:
    recovery_point_id: UUID
    command_id: UUID
    command_type: str
    request_fingerprint: str
    workspace_id: UUID
    database_path: Path
    recovery_root: Path
    schema_revision: Literal["0004_gate3_protected_crud"]

@dataclass(frozen=True)
class VerifiedRecoveryPoint:
    recovery_point_id: UUID
    command_id: UUID
    generation: int
    database_sha256: str
    snapshot_count: int
    snapshot_manifest_hash: str
    manifest_bytes: bytes
    promoted_slot: Literal["current"]

@dataclass(frozen=True)
class DomainMutation:
    entity_type: str
    entity_id: UUID
    operation: str
    expected_row_version: int | None
    before_snapshot: bytes | None
    after_snapshot: bytes | None
    changed_fields: tuple[str, ...]
    event_type: str
    event_payload: bytes

@dataclass(frozen=True)
class DecisionReviewBundle:
    candidate_id: UUID
    candidate_row_version: int
    detector_id: str
    detector_version: str
    rule_id: str
    rule_config_fingerprint: str
    earlier_snapshot_id: UUID
    later_snapshot_id: UUID
    direction_rationale: bytes
    signals: bytes
    input_observation_ids: tuple[UUID, ...]
    existing_memberships: tuple[UUID, ...]
    existing_relation_ids: tuple[UUID, ...]
```

`DecisionReviewBundle` is a read projection over immutable Gate 2 evidence and existing Gate 3 facts. It is not a new database entity, does not duplicate research text or absolute paths, and cannot mutate a candidate.
