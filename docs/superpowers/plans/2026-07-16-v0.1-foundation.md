# Research Workspace v0.1 Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the approved, offline-capable Windows `v0.1-foundation` desktop shell with locked contracts, domain/schema baseline, migrations, sample data, configuration, privacy-safe logging, independent Qt `.ui` pages, and AC1–AC16 verification—without implementing any post-foundation capability.

**Architecture:** `app.py` delegates to a composition root in `bootstrap.py`. Presentation loads Designer-owned `.ui` files and calls application queries/services; domain code owns immutable values and policy functions; infrastructure owns configuration, logging, SQLite, migrations, repositories, and seed data. Dormant parser/task/agent compatibility is represented only by protocols, schemas, tables, and pure policy validation.

**Tech Stack:** Python 3.12, uv, PySide6, SQLite, SQLAlchemy 2, Alembic, platformdirs, jsonschema with format checking, rfc8785, pytest, pytest-qt, pytest-socket, pip-licenses.

## Global Constraints

- Implement only `v0.1-foundation` from `docs/superpowers/specs/2026-07-15-foundation-design.md` at commit `dc9d62d` plus its approved status update.
- Do not implement document parsing, file monitoring, CRUD, AI, semantic search, context recovery, or Agent/task execution.
- Python metadata is exactly `>=3.12,<3.13`; use a project `.venv` and committed `uv.lock`.
- Follow the locked production tree exactly. No undeclared production modules.
- Every long-lived page has a separate Designer `.ui`; Python must not imperatively build page layouts.
- Original research/design files remain byte-identical and read-only from application workflows.
- Logs exclude paper bodies, extracted text, model inputs, secrets, and source-file contents.
- GPL, AGPL, unknown, and unreviewed commercial-only distribution dependencies are denied. LGPL packages require an explicit allowlist and notice.
- Every production behavior begins with a test observed failing for the intended reason.
- A task is not complete until its focused tests and the accumulated suite pass.
- Any required normative contract/schema/tree/scope change stops execution for user clarification.

```python
import json
import re
import subprocess
import sys
from pathlib import Path


ROOT = Path(__file__).resolve().parents[2]
PERMISSIVE_TOKENS = {"MIT", "BSD", "Apache", "ISC", "PSF", "Python Software Foundation"}
DENIED_TOKENS = {"GPL", "AGPL", "UNKNOWN"}
LGPL_ALLOWLIST = {"PySide6", "PySide6_Addons", "PySide6_Essentials", "shiboken6"}


def commercial_policy_violations(rows, permissive_tokens, lgpl_allowlist, denied_tokens):
    violations = []
    for row in rows:
        name = row["Name"]
        license_text = (row.get("License") or "UNKNOWN").strip()
        if name in lgpl_allowlist and "LGPL" in license_text:
            continue
        if any(token.casefold() in license_text.casefold() for token in permissive_tokens):
            continue
        has_gpl = "AGPL" in license_text.upper() or re.search(r"(?<!L)GPL", license_text.upper())
        if has_gpl or license_text.upper() in denied_tokens:
            violations.append((name, row["Version"], license_text))
        else:
            violations.append((name, row["Version"], f"UNREVIEWED: {license_text}"))
    return violations


def test_locked_dependencies_have_reviewed_commercial_licenses():
    rows = json.loads(subprocess.check_output([
        sys.executable, "-m", "piplicenses", "--format=json", "--with-license-file"
    ]))
    violations = commercial_policy_violations(
        rows,
        permissive_tokens=PERMISSIVE_TOKENS,
        lgpl_allowlist=LGPL_ALLOWLIST,
        denied_tokens=DENIED_TOKENS,
    )
    assert violations == []
    notices = (ROOT / "THIRD_PARTY_NOTICES.md").read_text(encoding="utf-8")
    assert "uv run pip-licenses --format=markdown --with-urls --with-license-file" in notices
    for row in rows:
        assert row["Name"] in notices
        assert row["Version"] in notices
```

Keep `commercial_policy_violations` inside the test because the policy is a release check, not a production module. It checks the LGPL package allowlist before scanning GPL tokens so `LGPL` is not misclassified as `GPL`; it fails closed on empty/unknown/unrecognized license strings and reports package, version, and observed license.
