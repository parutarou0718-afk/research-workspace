# UI 设计规范 v0.1

## 1. 视觉方向

采用已选定的第二套方案：浅色、蓝紫主色、少量青绿色状态色，现代 SaaS 风格。整体专业、可信、轻盈，不做暗色赛博或普通管理后台。

参考图：`assets/ui_reference.png`

## 2. 桌面基准

- 设计画布：1440 × 900
- 最小窗口：1180 × 720
- 左侧导航：88 px（收起）/ 220 px（展开，后续）
- 页面外边距：24 px
- 卡片间距：16 px
- 圆角：12 px
- 主卡片阴影：0 6 24 rgba(45, 65, 120, 0.08)

## 3. 色彩

- Primary: #4F67F5
- Primary Soft: #EEF1FF
- Violet: #7B61FF
- Teal: #25B8A6
- Success: #2FAE75
- Warning: #F0A23A
- Danger: #E85B64
- Text Main: #17213A
- Text Secondary: #667085
- Border: #E6EAF2
- Surface: #FFFFFF
- Background: #F6F8FC

## 4. 页面

### 总览

布局：顶部标题 + 4 个统计卡 + AI 建议/快速 Idea + 投稿总览 + 最近活动/本周专注。

主按钮：`立即整理`。快速 Idea 入口必须始终可见。

### 论文上下文恢复

这是视觉和产品核心。

顶栏：返回、论文标题、状态标签、立即整理、生成上下文、查看版本。

主体左侧固定十段式恢复卡；右侧为“证据与来源”和版本历史。

每条 AI 结论右侧提供：来源、置信度、确认、纠正。

### 投稿与知识关联

上半部为四列投稿看板；下半部为知识关系图和返修模式。

知识图谱首版只做简洁关系展示，不做复杂图数据库编辑器。

## 5. 交互状态

- AI 候选：淡紫底 + `AI 建议` 标签
- 用户确认：浅绿底 + `已确认`
- 待确认：淡黄边框
- 原文引用：白底 + 左侧蓝色引用线
- 错误：行内说明，不使用阻塞弹窗，除非数据删除

## 6. 首版导航

- 总览
- 论文
- Idea 库
- 投稿
- 会议
- 基金
- 设置

会议和基金可显示 `Coming Soon`，但必须保留数据模型入口。

## 7. UI 文件说明

`ui/research_workspace_main.ui` 是 Qt Designer XML 原型，包含：

- 左侧导航
- 总览页
- 统计卡
- AI 建议
- 快速 Idea
- 投稿表
- 最近活动

Codex 应将其转换为 Python 代码或使用 `QUiLoader`，然后继续实现论文上下文页和投稿页。禁止在生成后的 Python 文件中手改长期维护的布局；UI 结构应回写到 `.ui` 或拆成独立组件。
