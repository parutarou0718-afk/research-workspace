# 科研工作台 PRD v0.1

## 1. 产品定义

**产品名称**：科研工作台 / Research Workspace  
**定位**：以持续产出为核心的 AI 科研工作台。  
**核心价值**：帮助研究者快速恢复论文上下文，复用已有知识，并管理投稿过程。

产品不是论文写作机器人，也不是美化版 Excel。投稿记录、会议和基金只是外围流程；真正的核心是“研究记忆恢复”。

## 2. 目标用户

- 博士生、高校教师、科研人员
- 同时维护多篇论文、多个版本和多条研究线索的用户
- 经常经历投稿、返修、参会、拆稿、并稿和长期搁置后重启写作的用户

## 3. 核心问题

1. 同一论文存在大量 DOCX 版本，用户无法快速判断版本关系和修改内容。
2. 论文被搁置数月后，用户难以恢复当时的论点、结构、取舍和未使用材料。
3. Idea 分散在论文、读书笔记、会议 PPT、聊天记录和临时语音中。
4. 一个 Idea 可被多篇论文重复引用，一篇论文也可能拆分、合并或改题。
5. 投稿状态和返修截止日期需要一览，但表格只能记录状态，无法恢复写作上下文。

## 4. 产品成功标准

核心验证指标：

- 用户导入一篇论文的多个版本和相关 Idea 后，能够在 3 分钟内获得可核验的上下文恢复卡。
- 上下文恢复卡至少正确呈现：当前论点、结构、上次修改、重要删除内容、可复用 Idea、证据来源。
- 用户认为该结果显著优于自己手动翻阅文件夹和 Excel。

辅助指标：

- 每周至少 3 次快速记录 Idea。
- 每篇活跃论文至少关联 3 条 Idea 或材料。
- 返修时从打开项目到明确下一步的时间下降 50% 以上。

## 5. MVP 功能

### 5.1 总览

显示：
- 返修中、待投稿、会议临近、基金 DDL 数量
- AI 今日建议
- 快速记录 Idea
- 投稿总览
- 最近活动
- 本周专注

会议和基金在 MVP 可仅显示手动记录的数据；不做自动推荐。

### 5.2 论文项目

- 新建论文项目
- 导入 DOCX、PDF、PPTX、TXT、Markdown
- 指定当前工作版本
- 自动生成候选版本关系
- 用户可纠正版本归属
- 论文可拆分、合并，但仅继承用户确认的相关内容
- 查看关联 Idea、会议、投稿和版本历史

### 5.3 Idea 库

- 低摩擦输入：文字优先，预留语音入口
- 所有被系统判断为重要且从新版本中消失的完整论点，自动生成候选 Idea
- 一个 Idea 可关联多篇论文
- Idea 不要求必须归属于论文
- Idea 可来源于书籍、论文、会议、聊天、临时思考
- 状态：未使用、已使用、搁置、已归档
- 所有来源可追溯

### 5.4 上下文恢复（核心）

固定输出结构：

1. 论文目前在讲什么
2. 核心论点
3. 当前结构
4. 上次修改
5. 已删除的重要内容
6. 可复用 Idea
7. 最近新增的相关材料
8. 关联会议与投稿状态
9. 下一步建议
10. 证据与来源

规则：
- 每条 AI 判断必须引用文件、版本、章节或笔记来源。
- AI 推断和用户原文必须视觉区分。
- 不允许直接修改原始论文。

### 5.5 投稿管理

提供投稿现状一览表：
- 论文
- 期刊/会议
- 投稿日期
- 当前阶段
- 截止日期
- 最新动作
- 关联版本
- 审稿意见附件

支持状态：准备中、待投稿、编辑初审、外审中、返修中、已接受、已拒绝、撤稿、长期无回复。

### 5.6 整理机制

两种触发方式均需支持：
- 用户点击“立即整理”
- 定期整理任务

定期任务只扫描新增或发生变化的文件，不得每次全量遍历全部历史文件。

## 6. 非 MVP / DLC

### 6.1 邮箱投稿助手

读取邮件、识别稿件、提取状态和截止时间、下载审稿附件并更新投稿记录。

### 6.2 会议增强

管理摘要、PPT、发言稿、现场意见及会后论文修改。

### 6.3 基金推荐

综合论文、未使用 Idea、计划研究方向和能力边界，推荐基金题目与方向；联网匹配真实基金机会。

### 6.4 返修 Agent

拆分审稿意见、定位论文章节、检索可用材料、生成返修任务表。

### 6.5 多 Agent

未来可拆分文档、知识、返修、投稿、会议、基金 Agent。MVP 采用统一调度器和模块化工具，接口保持兼容。

### 6.6 OCR、移动端、云同步

作为独立连接器和 Provider 扩展。

## 7. 不可妥协的产品原则

1. 原始文件只读，不得被 AI 覆盖。
2. AI 结论必须可追溯。
3. 原文、AI 提取、AI 建议、用户确认严格区分。
4. 自动操作可撤销，并保留审计日志。
5. 数据可导出，避免平台锁定。
6. 无 AI 时仍可浏览、检索和手动维护。
7. 低摩擦输入，不要求用户维护复杂标签体系。
8. 语义相似不得自动等同于论点相同。
9. 文件解析、模型、检索、连接器和任务执行均须可替换。

## 8. 典型用户流程

### 流程 A：恢复旧论文

1. 用户打开论文项目。
2. 点击“恢复上下文”。
3. 系统读取当前版本、历史版本、Idea、笔记、会议和投稿记录。
4. 检索新增相关材料。
5. 生成固定结构的恢复卡和来源引用。
6. 用户确认或纠正候选关联。
7. 用户从“下一步建议”进入写作。

### 流程 B：新版本整理

1. 用户将新 DOCX 放入监控目录或手动导入。
2. 系统通过文件指纹和内容相似度识别候选项目。
3. 生成版本差异。
4. 发现疑似被删除的重要论点，自动进入 Idea 候选库。
5. 用户确认关联。

### 流程 C：投稿返修

1. 用户把投稿状态改为返修中，关联审稿意见。
2. 系统恢复该论文上下文。
3. 系统逐条整理审稿意见并查找可复用 Idea。
4. 生成返修任务清单。

## 9. 数据与隐私

支持三种模式：
- 本地：数据、文件、模型均在本机
- 混合：数据本地，仅发送必要片段至 API
- 云端：未来扩展

调用云模型前必须显示：发送对象、片段范围、Provider 和是否记录日志。

## 10. 首版验收标准

- 可运行于 Windows 10/11。
- 可创建论文和 Idea，导入 DOCX/PDF/PPTX。
- 可生成投稿总览。
- 可手动和定期触发增量整理。
- 可生成带来源引用的上下文恢复卡。
- 可导出 JSON/Markdown/CSV 和数据库备份。
- 所有 AI 生成操作可撤销。
